## ⚠️ Aviso sobre Proyecto de Portafolio

**Este es solo un proyecto de demostración/portafolio.** ALLFLIX NO transmite contenido de video real y no es un servicio de streaming funcional.

Este proyecto representa un **concepto idealizado** - imagine un escenario donde una superproductora adquirió los derechos de todo el contenido de entretenimiento y lo puso a disposición en una única plataforma unificada. Es un "escenario de ensueño" para fines educativos y de portafolio.

## 🎯 Descripción General

**ALLFLIX** es un proyecto de portafolio profesional full-stack que demuestra la implementación de una interfaz moderna de plataforma de streaming. Construido con tecnologías de vanguardia, presenta desarrollo frontend avanzado, arquitectura backend, diseño de base de datos e integraciones de API.

**Capacidades principales:**
- Interfaz moderna y responsiva inspirada en las principales plataformas de streaming
- Sistema completo de autenticación con gestión segura de sesión
- Integración con API TMDB para 500,000+ películas y series
- Funcionalidad avanzada de búsqueda y filtrado
- Gestión de perfil de usuario y características de lista de favoritos
- Optimizado para rendimiento con Next.js 16 App Router
- Estructura de código lista para producción con mejores prácticas

## 🚀 Inicio Rápido

### Requisitos Previos

Antes de comenzar, asegúrese de tener:
- **Node.js 20+** instalado ([Descargar](https://nodejs.org/))
- **Base de datos PostgreSQL** (local o proveedor en la nube como Neon, Supabase, Railway)
- **Clave de API TMDB** (gratuita en [https://www.themoviedb.org/settings/api](https://www.themoviedb.org/settings/api))

### Opción 1: Desarrollo Local (Recomendado)

```bash
# 1. Clone el repositorio
git clone https://github.com/TechBeme/allflix.git
cd allflix

# 2. Instale las dependencias
npm install

# 3. Configure las variables de entorno
cp .env.example .env.local
# Edite .env.local con sus credenciales (vea sección Configuración)

# 4. Configure la base de datos
npm run db:generate    # Genera el cliente Prisma
npm run db:push        # Envía el schema a la base de datos
npm run db:seed        # (Opcional) Llena con datos de ejemplo

# 5. Ejecute el servidor de desarrollo
npm run dev
```

Abra [http://localhost:3000](http://localhost:3000) para ver la aplicación.

### Opción 2: Build de Producción

```bash
# Build para producción
npm run build

# Inicia servidor de producción
npm run start
```

### Opción 3: Gestión de Base de Datos

```bash
# Abre Prisma Studio (GUI de Base de Datos)
npm run db:studio

# Crea una nueva migración
npm run db:migrate

# Vea la base de datos en el navegador en http://localhost:5555
```

## ⚙️ Configuración

### Variables de Entorno

Cree un archivo `.env.local` en el directorio raíz:

```bash
# Base de Datos (PostgreSQL)
# Ejemplo para Neon: postgresql://user:password@ep-xxx.us-east-2.aws.neon.tech/allflix?sslmode=require
# Ejemplo para local: postgresql://postgres:password@localhost:5432/allflix
DATABASE_URL="postgresql://user:password@host:5432/allflix"

# API TMDB (obtenga clave gratuita en https://www.themoviedb.org/settings/api)
TMDB_API_KEY="su_clave_api_tmdb_aqui"
TMDB_API_BASE_URL="https://api.themoviedb.org/3"

# Autenticación (genere cadenas aleatorias seguras)
BETTER_AUTH_SECRET="su-clave-secreta-minimo-32-caracteres"
BETTER_AUTH_URL="http://localhost:3000"

# Next.js
NEXT_PUBLIC_APP_URL="http://localhost:3000"
NODE_ENV="development"
```

### Opciones de Base de Datos

El proyecto soporta múltiples proveedores PostgreSQL:

| Proveedor | Tipo | Mejor Para |
|-----------|------|------------|
| **[Neon](https://neon.tech/)** | Serverless | Auto-escalado, tier gratuito generoso |
| **[Supabase](https://supabase.com/)** | Administrado | PostgreSQL + extras (auth, storage) |
| **[Railway](https://railway.app/)** | Administrado | Deploy simple, precios justos |
| **[Vercel Postgres](https://vercel.com/storage/postgres)** | Serverless | Integrado con alojamiento Vercel |
| **PostgreSQL Local** | Auto-alojado | Control total, costos cero |

### Comandos Disponibles

```bash
# Desarrollo
npm run dev              # Inicia servidor de desarrollo (http://localhost:3000)
npm run build            # Build para producción
npm run start            # Inicia servidor de producción
npm run lint             # Ejecuta ESLint
npm run lint:fix         # Corrige errores de linting automáticamente
npm run type-check       # Ejecuta verificación de tipos TypeScript

# Base de Datos
npm run db:generate      # Genera el Prisma Client
npm run db:push          # Envía cambios del schema a la base de datos
npm run db:migrate       # Crea y aplica migraciones
npm run db:seed          # Llena base de datos con datos de ejemplo
npm run db:studio        # Abre Prisma Studio (GUI de base de datos)

# Pruebas
npm run test             # Ejecuta todas las pruebas
npm run test:watch       # Ejecuta pruebas en modo watch

# Análisis
npm run analyze          # Analiza tamaño del bundle
```
